# Definition of Ready (DoR)

A story is **Ready** when:

- [ ] Acceptance criteria are written and testable
- [ ] Scope is set: Mandatory or Bonus
- [ ] Subsystem is set (Parser/Expansion/Builtins/Execution/Pipes/Redirections/Signals/Testing/Docs)
- [ ] Dependencies are known (blocked items identified)
- [ ] A short test plan exists (3–5 bullets or example commands)
- [ ] The work is small enough to be reviewed (or has been split)
